# Knowledge Market — 改善プラン / 立ち上げドキュメント

作成日: 2026-02-20

## 1. サマリー（現状評価）
**強み**
- AIエージェントが自律購入できるマーケットという“差別化軸”が強い
- MCP / CLI / API が揃っており、開発者体験が良い
- Solana決済 + 自動分配で「プロトコル感」が出せる

**惜しい点（改善余地）**
- AIが“即使える形”の知識フォーマットがない
- 初期出品が少ないと空気になりやすい
- 購入までの心理的ハードルが高い（Solana/ウォレット）
- 「なぜこのサイトで買うのか」の一点突破メッセージが薄い

---

## 2. コア方針（推奨）
**Agent-firstに絞る（最初の1〜2ヶ月）**
- 人間向け売買は後回し
- 価値は「AIが自律購入→即タスク進行」
- MCP/CLI の磨き込みを最優先

**初期フェーズは“クローズドβ”が最適**
- 品質・価格・需要の当たりを早く見つけられる
- 小さく回して“勝ちパターン”を作れる

---

## 3. 初期ターゲット
**最初の売り手**
- AI開発者 / 自動化エンジニア / 生成AI運用者
- 「自分のノウハウをテンプレ化できる人」

**最初の買い手**
- エージェント開発者 / 開発速度を上げたい人
- 企業のAI導入を担当する実務者

---

## 4. 売れる知識の「型」
AIが即使えるように、**構造テンプレ**を必須にする。

### 4.1 テンプレ（最小）
```
# Title
# Goal (何ができるようになるか)
# Preconditions (前提・環境)
# Steps (再現手順)
# Output (具体的な出力例)
# Failure Cases (失敗例 / 罠)
# Verification (成功判定)
# Usage Prompt (AIエージェント向けの投入文)
```

### 4.2 “売れる”カテゴリ（初期4つ）
1) **Prompt**：タスク特化の高再現プロンプト
2) **MCP**：エージェント連携の設定・ツール定義
3) **Dataset**：分類済み・加工済みの実務データ
4) **Playbook**：運用手順 / 失敗回避 / 判断基準

---

## 5. 初期出品（デモ）案
最低 3〜5件を最初に揃える（空マーケット回避）。

**例**
- 「Rails業務SaaSのCSV/XLSXインポート設計テンプレ」
- 「AIエージェント向けの要件整理プロンプトセット」
- 「Supabase×RLSの安全な設計ガイド」
- 「Solana決済の検証手順プレイブック」
- 「運用ダッシュボードのKPI設計テンプレ」

---

## 6. 価格・決済ハードル設計
- **初期は低価格帯を用意**（例: 0.1 / 0.3 / 0.5 SOL）
- 初回体験のため**無料 or 0.1 SOL枠**を置く
- 「AIが即使える」という価値を前面に出し、価格納得感を作る

---

## 7. UI/UX 改善（P2）
Plans.md にある内容を最優先で実施。

- EVM対応をUIから隠す
- i18n切替をUIから隠す
- request listing をUIから隠す

→ “未完成感”を消して、シンプルな価値だけ見せる

---

## 8. メッセージ設計（必須）
トップメッセージで一撃を狙う。

**例**
- “The first marketplace where AI agents autonomously buy human knowledge.”
- “Upload real human expertise. Agents can purchase and use it instantly.”

---

## 9. リリースまでの小さなロードマップ

**Week 1**
- テンプレ導入
- 初期出品 3〜5件
- UIの未対応機能を隠す

**Week 2**
- クローズドβで実運用テスト
- 初回購入デモ動画/GIF

**Week 3**
- 公開（Vercel/GitHub）
- SNS + 開発者コミュニティへ投稿

---

## 10. KPI（最初の1ヶ月）
1) 自律購入の成功回数
2) 初期出品の購入率
3) 購入後フィードバック
4) 出品継続率

---

## 11. 次のアクション提案
1) **テンプレ仕様を実装**（UI/DB/API）
2) **初期出品を作成**（3〜5件）
3) **LPのメッセージを強化**
4) **デモ動画を作成**
