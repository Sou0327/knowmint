# KnowMint 法的リスク調査報告書

> 作成日: 2026-02-23
> 注意: 本報告書は公開情報に基づく調査であり、法的助言ではありません。正式な判断には暗号資産に詳しい弁護士への相談が必要です。

## 前提: KnowMint のビジネスモデル

```
買い手 --[SOL/USDC]--> 売り手のウォレット（P2P直接送金）
            ↑
       KnowMint は tx_hash を検証・記録するだけ
       秘密鍵を持たない、資産を預からない（ノンカストディアル）
```

- **提供サービス**: 人間の暗黙知・体験知をAIエージェントに販売するマーケットプレイス
- **決済手段**: 暗号資産（Solana SOL/USDC、将来的にEVM対応）
- **決済方式**: ノンカストディアル P2P 直接送金
- **運営の関与**: tx_hash の検証と記録のみ。秘密鍵を持たず、資産を預からない
- **将来予定**: スマートコントラクトによるプロトコル手数料の自動分配

## 1. 暗号資産交換業（資金決済法63条の2）

### 交換業の定義

暗号資産交換業に該当する行為は以下の3つ:

1. 暗号資産の売買又は他の暗号資産との交換
2. 上記の **媒介**、取次ぎ又は代理
3. 暗号資産の **管理（カストディ）**

### KnowMint の該当性分析

| 行為 | 該当？ | 理由 |
|------|--------|------|
| 暗号資産の売買・交換 | No | 運営は暗号資産の売買を行っていない |
| カストディ（管理） | No | 秘密鍵を持たない。JCBI照会で金融庁がノンカストディの非該当を認めた前例あり |
| **媒介** | **グレー** | 下記参照 |

### 「媒介」該当性 — 最大の論点

KnowMint は **ナレッジ（デジタルコンテンツ）の売買** を仲介している。その **決済手段** として暗号資産が使われる。

- **KnowMint の主張**: 「ナレッジ売買の媒介であって、暗号資産売買の媒介ではない」
- **反論リスク**: 金融庁が「実質的に暗号資産の移転を促進するプラットフォーム」と見なす可能性

これは NFT マーケットプレイスと同じ構造。金融庁ガイドラインでは「個別事例ごとに実態に即して実質的に判断するべき」とされている。

**評価**: 現行の設計では交換業に該当しない可能性が高いが、確定的ではない。

### カストディ業務に関する金融庁の前例

2024年10月、金融庁はJCBI（ジャパン・コンテンツ・ブロックチェーン・イニシアティブ）のPassWallet(R)サービスについて、グレーゾーン解消制度に基づく照会に回答し、カストディ業務該当性を否定した。これはカストディ業務該当性に関してグレーゾーン解消制度で金融庁が回答を行った初の事例。

ただし本回答は当該サービスに限定されたものであり、「必ずしも他のサービスにそのまま及ぶものではない」と明記されている。

**参考**: [PwC: カストディ業務該当性に関する金融庁回答](https://www.pwc.com/jp/ja/knowledge/news/legal-news/legal-20241028-2.html)

## 2. 2025年改正資金決済法 — 新「仲介業」との関係

2025年6月成立の改正資金決済法で、**「電子決済手段・暗号資産サービス仲介業」** が新設された。

### 新仲介業の要件

| 項目 | 仲介業の要件 | KnowMint |
|------|-------------|----------|
| 前提 | 暗号資産交換業者の **委託を受けて** 媒介 | 委託関係なし |
| 所属制 | 必須（特定の交換業者に所属） | 該当しない |
| KYC | 不要（特定事業者ではない） | — |
| 財務規制 | なし（利用者資産を預からないため） | — |

### 評価

KnowMint は暗号資産交換業者から委託を受けていないため、この新カテゴリにも該当しない。逆に言えば、軽量な仲介業登録で合法化する道もない。

**参考**: [BUSINESS LAWYERS: 2025年改正資金決済法の概要](https://www.businesslawyers.jp/articles/1476), [ZeLo: 暗号資産サービス仲介業の概要](https://zelojapan.com/en/lawsquare/59522)

## 3. 金商法移行（2028年施行見込み）

### 概要

金融審議会は2025年12月に報告書を公表。暗号資産を資金決済法から金融商品取引法に移行する方針。

- **2026年** 通常国会に金商法改正案提出
- **2028年1月** 施行見込み
- 暗号資産交換業者には第一種金融商品取引業に相当する規制
- インサイダー取引規制、発行者への年1回の情報開示義務
- 分離課税の適用（施行翌年から）

### KnowMint への影響

- KnowMint は交換業者ではないため、直接の規制対象にはならない可能性
- ただし「暗号資産を決済手段として使うプラットフォーム」への規制が今後追加されるリスクはある
- ノンカストディアル・DEXの規制は「将来の検討課題」として先送りされている状態

**評価**: 直接的な影響は低〜不明だが、法改正の動向監視が必要。

**参考**: [日経: 仮想通貨を金商法で規制へ](https://www.nikkei.com/article/DGXZQOUB109M10Q5A211C2000000/), [森・濱田松本: 暗号資産規制の見直し動向](https://www.morihamada.com/ja/insights/newsletters/121331)

## 4. プロトコル手数料の法的リスク

### 現行の予定

> 将来的にスマートコントラクト自動分配でプロトコル手数料を徴収予定（CLAUDE.md記載）

### 設計パターン別リスク

| 設計パターン | リスク | 解説 |
|-------------|--------|------|
| **分配型**: スマコンが買い手→売り手に送金し、手数料を運営ウォレットに自動分配 | **低** | 運営は資産を一時的にも管理しない |
| **受取転送型**: 運営が一旦受け取って売り手に送金 | **高** | カストディ + 資金移動業に該当しうる |
| **エスクロー型**: スマコンが一時保持 | **中** | スマコンの管理者が誰かで判断が分かれる |

### 推奨

- **分配型（パターン1）を採用** — 現行の設計方針に合致
- スマートコントラクトの管理者権限（upgrade可能か、pause可能か）次第で「実質的な管理」と見なされるリスクあり
- Immutable なスマコン設計が最もリスクが低い

## 5. 特定商取引法・消費者保護法

### 暗号資産とは別の問題

デジタルコンテンツ販売プラットフォームとして、以下の法律への対応が **確実に必要**。

### 5.1 特定商取引法（通信販売）

| 義務 | 内容 | 対応状況 |
|------|------|---------|
| 販売者の表示義務 | 名称、住所、電話番号、返品特約等 | 未対応 |
| 広告規制 | 誇大広告の禁止等 | 部分対応 |
| 契約に関する表示 | 価格、支払方法、引渡時期等 | 部分対応 |

**注意**: KnowMint は「プラットフォーム」であり売主は出品者。出品者に表示義務を課す仕組みが必要。

### 5.2 取引DPF消費者保護法（2022年5月施行）

「取引デジタルプラットフォームを利用する消費者の利益の保護に関する法律」

| 義務 | 内容 |
|------|------|
| 販売者の身元確認努力義務 | 出品者の身元を一定程度確認する |
| 苦情処理体制 | 消費者からの苦情を受け付け処理する体制 |
| 販売者情報の開示請求への対応 | 消費者からの開示請求に応じる |

### 5.3 利用規約

| 項目 | 状態 |
|------|------|
| 免責事項 | 未確認 |
| 準拠法・管轄裁判所 | 未確認 |
| 禁止事項 | 未確認 |
| 返金ポリシー | 未確認（デジタルコンテンツのため特に重要） |

## 6. 犯収法（KYC / マネーロンダリング対策）

- 暗号資産交換業に該当しなければ、犯収法上の「特定事業者」ではない → **KYC義務なし**
- ただし FATF のトラベルルールは年々厳格化
- 大規模プラットフォームには自主的な KYC 導入が望ましい
- 特に高額取引（数十万円以上）が頻発する場合、当局から注目される可能性

## 総合リスク評価

| リスク | レベル | 備考 |
|--------|--------|------|
| 暗号資産交換業該当（現行） | **中低** | ノンカストディ + ナレッジ売買媒介。ただし金融庁判断は個別 |
| 金商法移行の影響（2028年） | **低〜不明** | 直接対象外だが法改正の動向監視が必要 |
| プロトコル手数料 | **設計次第** | 分配型スマコンなら低リスク |
| 特商法・消費者保護 | **確実に対応必要** | マーケットプレイスとしての表示義務 |
| KYC / マネロン | **現時点では低** | 規模拡大時に自主対応推奨 |

---

## 推奨アクションアイテム

### 短期（優先度高）

1. **特商法/DPF消費者保護法の対応** — 利用規約、出品者の表示義務、苦情処理体制。弁護士なしでもある程度対応可能
2. **利用規約の整備** — 免責事項、準拠法、返金ポリシー

### 中期（法的リスク軽減）

3. **暗号資産に詳しい弁護士にグレーゾーン解消制度の利用を相談** — JCBIのように金融庁に直接照会できる。最も確実
4. **プロトコル手数料のスマコン設計** — エスクロー型は避け、分配型に。Immutable設計推奨

### 長期（監視）

5. **金商法改正の動向を定期的にウォッチ** — 2026年の国会審議、2028年施行
6. **規模拡大時のKYC自主導入** — FATF トラベルルール厳格化に備える
